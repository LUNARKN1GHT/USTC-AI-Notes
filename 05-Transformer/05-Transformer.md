## 主线

本章的任务是解决**序列建模中全局依赖难以高效捕获**的问题，核心路径分三步：

1. **问题背景** — [[5.1-Transformer简介与注意力机制|RNN]] 受梯度消失困扰，长距离依赖难以建模；[[5.1-Transformer简介与注意力机制|LSTM]] 通过门控机制部分缓解，但训练仍是串行，效率低。
2. **注意力机制** — 允许模型在生成每个输出时，直接参考输入序列的任意位置，打破逐步传递的局限。[[5.1-Transformer简介与注意力机制|自注意力机制]]进一步令序列中每个位置同时充当查询（Q）、键（K）和值（V）。
3. **Transformer 架构** — [[5.2-Transformer 的编码与解码器|编码器-解码器]]结构以多头自注意力为核心，辅以位置编码、[[残差连接与层归一化]]，实现可并行化的全局序列建模。[[5.3-Transformer 的典型应用]]覆盖 NLP、视觉、多模态等领域。

> **关键公式**：$\text{Attention}(Q,K,V) = \text{Softmax}\!\left(\dfrac{QK^T}{\sqrt{d_k}}\right)V$。缩放因子 $\frac{1}{\sqrt{d_k}}$ 防止内积过大导致梯度消失。

## 关键概念

| 概念 | 一句话 |
| :-- | :-- |
| [[5.1-Transformer简介与注意力机制\|RNN]] | 时间步参数共享，顺序建模；梯度消失是主要瓶颈 |
| [[5.1-Transformer简介与注意力机制\|LSTM]] | 遗忘门 / 输入门 / 输出门，缓解 [[RNN]] 的长程梯度消失 |
| [[5.1-Transformer简介与注意力机制\|注意力机制]] | 动态加权输入序列各位置，生成上下文向量 $c$ |
| [[5.1-Transformer简介与注意力机制\|自注意力机制]] | 序列每位置同时作为 Q/K/V，一次并行计算所有位置的关联 |
| [[5.1-Transformer简介与注意力机制\|多头注意力]] | 并行多组自注意力，在不同子空间捕获多种特征模式 |
| [[5.2-Transformer 的编码与解码器\|位置编码]] | 以正弦/余弦函数注入序列顺序信息，弥补自注意力无位置感知 |
| [[5.2-Transformer 的编码与解码器\|残差连接与层归一化]] | 稳定深度网络训练，缓解梯度消失/爆炸 |
